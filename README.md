# pkgto-web

A WebAssembly-powered GTO preflop equity analyzer built on [pkcore](https://crates.io/crates/pkcore).
Enter your hole cards and a villain range, and get per-hand matchup equity plus combined odds ‚Äî entirely in the browser, no server required.

## Demo

## How It Works

The Rust crate compiles to WebAssembly via [wasm-pack](https://rustwasm.github.io/wasm-pack/).
It exposes a single function to JavaScript:

```js
analyze_gto(hero: string, villain_range: string) ‚Üí JSON string
```

pkcore embeds 812,175 precomputed heads-up preflop matchup results (from `hups.bin`) directly
into the WASM binary via `include_bytes!`, so all lookups happen in-memory with no network
requests or database connections.

## Prerequisites

- [Rust](https://rustup.rs/) with the `wasm32-unknown-unknown` target:
  ```sh
  rustup target add wasm32-unknown-unknown
  ```
- [wasm-pack](https://rustwasm.github.io/wasm-pack/installer/):
  ```sh
  cargo install wasm-pack
  ```
- Python 3 (for the dev server, or any static file server)

## Quick Start

```sh
# Build the WASM module and serve at http://localhost:8080
make serve
```

Then open [http://localhost:8080](http://localhost:8080) in your browser.

## Hand Notation

Both ASCII and Unicode suit symbols are accepted.

| Format      | Example        |
|-------------|----------------|
| Unicode     | `K‚ôÝ K‚ô•`        |
| ASCII suits | `KsKh`         |
| Spaced      | `Ks Kh`        |

## Range Notation

Ranges follow standard poker shorthand:

| Notation   | Meaning                                   |
|------------|-------------------------------------------|
| `AA`       | Pocket aces (all suit combos)             |
| `66+`      | Pocket sixes through pocket aces          |
| `AKs`      | Ace-king suited                           |
| `AJs+`     | Ace-jack suited through ace-king suited   |
| `AJo+`     | Ace-jack offsuit through ace-king offsuit |
| `AKo`      | Ace-king offsuit                          |

Combine with commas: `66+,AJs+,KQs,AJo+,KQo`

## Size Notes

The WASM binary is approximately **15 MB** in a dev build because pkcore embeds the full
precomputed equity table (`hups.bin`). A real HTTP server serving with **gzip or brotli
compression** will typically reduce transfer size to around **5‚Äì7 MB**.

A release build (`make build-release`) applies `wasm-opt` and reduces the binary further.
